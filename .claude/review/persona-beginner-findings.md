## このペルソナを満足させるための改善TOP3

### 1. トップページに「このツールは何か」「どう使うか」を日本語で明示する（優先度: 最高）

現状のタイトル「Shopee Optimizer」を、例えば「Shopee出品かんたんツール」のような日本語名に変更する。サブタイトルも「AmazonのURLを入れるだけで、Shopee向けの商品ページと紹介動画を自動作成します」のように平易な一文にする。さらに、3ステップの使い方ガイド（「1. AmazonのURLをコピペ → 2. ボタンを押して待つ → 3. できた動画を確認」）をイラストまたはアイコン付きで表示する。

### 2. エラーメッセージを「原因 + 対処法」のセットにし、alert()からインラインUIに変更する（優先度: 高）

`alert()` を廃止し、画面内にエラーメッセージを表示するインライン通知に変更する。メッセージは「何が起きたか（原因）」と「どうすればいいか（対処法）」をセットにする。例: 「通信エラーが発生しました。インターネット接続を確認して、もう一度お試しください。」/ 「URLの形式が正しくありません。Amazonの商品ページのURLを貼り付けてください。」。技術的な内部メッセージ（batch_id等）はユーザーに見せない。

### 3. 専門用語を平易な日本語に置き換える（優先度: 高）

以下の用語を置き換える:
- 「タブ追加 / タブ削除」 → 「商品を追加 / 商品を削除」
- 「一括実行」 → 「作成スタート」
- 演出名 zoom/unbox/steam/condensation/pickup → 「ズームアップ / 開封風 / 湯気 / 水滴 / 手に取る」+ 各演出のサムネイルプレビュー
- 「全て確定してスプレッドシート反映」 → 「この内容で保存する」（確認ダイアログ付き）
- 「再生成メモ」 → 不要であれば削除、必要なら「メモ（任意）」にしてツールチップで用途説明
- ステップ名「画像テキスト英語化」→「画像の文字を翻訳中」等

## 次のPhaseへの引き継ぎ事項

1. **エラーハンドリングの全面的な見直しが必要**: 現在10箇所の `alert()` が使われており（行644, 776, 777, 1091, 1107, 1136, 1144, 1157, 1162, 1170）、全てユーザーフレンドリーなインライン通知に置き換える必要がある。特に `batch_idがありません` (行1162) のような内部エラーはユーザーに見せるべきではない。

2. **ログUIの扱いを検討**: `#log` 要素（行352-359）はターミナル風の黒背景UIで、一般ユーザーには不要。デフォルトで非表示にし、「詳細ログを見る」トグルで展開する形にすることを推奨。

3. **安心感を与える要素の追加が必須**: プライバシーポリシー、FAQ、問い合わせ先の導線がゼロ。山田花子のような「個人情報は大丈夫？」と不安を持つユーザーは、これらがないと離脱する可能性が高い。フッターにこれらのリンクを追加する設計を検討。

4. **オンボーディングフローの設計**: 初回アクセス時にツアーやウォークスルー（「ここにURLを入れてください」のようなステップバイステップガイド）を表示する仕組みの導入を推奨。

5. **確定操作の確認ダイアログ**: 「全て確定してスプレッドシート反映」（行583）は不可逆に見える操作のため、実行前に確認ダイアログを表示し、「何が起こるか」を明示すべき。現状は `finalizeBatch()` 関数（行1161-1174）にそのような確認ロジックがない。

6. **演出プレビューの必要性**: 動画エフェクト選択（行1030-1035）が英語テキストのドロップダウンのみ。各演出のサンプル動画やGIFプレビューを用意して、選択前に仕上がりイメージを伝える仕組みが望ましい。

7. **スマホ操作の最適化**: レスポンシブ対応はあるが（行526-537）、タッチ操作に特化したUI改善（ボタンサイズの拡大、スワイプでのタブ切り替え等）は未実装。山田花子はスマホメインユーザーであるため、この点の改善は重要。
